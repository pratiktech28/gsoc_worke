<div align="justify">
  <h2>🔱 Technical Overview & Engineering Design</h2>
  <p>
    This repository serves as the <strong>Automated Physics Validation Engine</strong> for the gprMax GSoC project. 
    The core objective is to bridge the gap between high-performance kernel optimization and physics integrity. 
    By leveraging a <strong>Dockerized CI/CD pipeline</strong>, we ensure that every code modification is 
    automatically verified against stringent electromagnetic wave standards.
  </p>
</div>

<hr />

<div>
  <h3>🚀 Automated NRMSE Physics Logic</h3>
  <p>
    Unlike traditional software testing, physics-based software requires a <strong>Quantitative Validation Layer</strong>. 
    Our system implements an automated Python-based kernel that executes the following workflow:
  </p>
  <ul>
    <li>
      <strong>Data Acquisition:</strong> The pipeline extracts raw wave data from the simulation output files 
      generated by the parallelized gprMax kernels.
    </li>
    <li>
      <strong>Error Quantification:</strong> We calculate the <strong>Normalized Root Mean Square Error (NRMSE)</strong> 
      to detect even the slightest deviations in wave propagation. This is critical when scaling to 
      <strong>100 Parallel Pads</strong>, as minor floating-point errors can accumulate.
    </li>
    <li>
      <strong>Threshold Enforcement:</strong> The CI/CD workflow is configured with a <strong>Strict-Fail Policy</strong>. 
      If the NRMSE exceeds 2%, the build is flagged, preventing inaccurate physics from reaching the production branch.
    </li>
  </ul>
</div>

<hr />

<div>

  <p>

  </p>
  
  <table width="100%">

      <td width="50%">

  </table>
